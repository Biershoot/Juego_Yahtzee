1. Introducción

El método de Montecarlo es una técnica computacional que utiliza la generación de números aleatorios para resolver problemas que pueden tener una solución determinista difícil de obtener analíticamente. Su nombre proviene del famoso casino de Montecarlo en Mónaco, haciendo alusión al carácter aleatorio de los juegos de azar. Fue formalizado por Stanislaw Ulam y John von Neumann durante la década de 1940 como parte del Proyecto Manhattan, y desde entonces se ha convertido en una herramienta fundamental en campos tan diversos como la física, las finanzas, la ingeniería y la inteligencia artificial (Metropolis & Ulam, 1949).

El principio fundamental del método consiste en realizar un gran número de simulaciones aleatorias de un proceso y, a partir de la distribución de los resultados obtenidos, estimar probabilidades, medias, varianzas y otras métricas estadísticas. La Ley de los Grandes Números garantiza que, a medida que el número de simulaciones crece, las estimaciones convergen hacia los valores teóricos reales.

En el presente trabajo se aplica el método de Montecarlo a la simulación del juego clásico Yahtzee, un juego de dados en el que dos jugadores lanzan cinco dados de seis caras en turnos alternados, disponiendo de hasta tres lanzamientos por turno. Los dados siguen una distribución de probabilidad uniforme discreta, donde cada cara tiene una probabilidad de 1/6 de aparecer en cada lanzamiento individual. El objetivo es analizar las distribuciones de probabilidad emergentes al lanzar múltiples dados simultáneamente y verificar, mediante simulación, que los resultados experimentales convergen hacia las probabilidades teóricas conocidas.

2. Metodología

2.1 Fundamento Teórico

El espacio muestral de un dado justo de seis caras es Ω = {1, 2, 3, 4, 5, 6}, donde cada resultado tiene probabilidad:

P(X = k) = 1/6, para k ∈ {1, 2, 3, 4, 5, 6}

Esta es la definición clásica de una distribución uniforme discreta. Al lanzar n dados independientes, el número total de resultados posibles es 6n. Para cinco dados (n = 5), existen 65 = 7,776 combinaciones posibles.

La suma de cinco dados sigue una distribución que se puede calcular mediante convolución de distribuciones uniformes. La media teórica de la suma es:

E[S₅] = 5 × E[X] = 5 × 3.5 = 17.5

Y la varianza:

Var[S₅] = 5 × Var[X] = 5 × 35/12 ≈ 14.583

Lo que da una desviación estándar teórica de σ ≈ 3.819.

La solución se diseñó como una aplicación web interactiva utilizando tecnologías estándar del lado del cliente (HTML5, CSS3 y JavaScript puro), lo que permite su ejecución directa en cualquier navegador moderno sin necesidad de instalaciones adicionales ni servidores. 2.4 Implementación del Generador Aleatorio

Cada dado se simula mediante la función Math.random() de JavaScript, que genera valores pseudo-aleatorios con distribución uniforme en el intervalo [0, 1). La conversión a un valor de dado (1-6) se realiza mediante:

function rollDie() {
    return Math.floor(Math.random() * 6) + 1;
}

Esta función implementa la transformación inversa de una distribución uniforme discreta. Math.random() utiliza el algoritmo xoshiro256** en los motores JavaScript modernos (V8, SpiderMonkey), que proporciona un periodo suficientemente largo (2256 - 1) para simulaciones estadísticas (Blackman & Vigna, 2021).

2.5 Algoritmo de Simulación Montecarlo

El procedimiento de simulación sigue estos pasos:

  1. Se define el número de iteraciones N (configurable por el usuario).
  2. Para cada iteración i de 1 a N:
    1. Se lanzan 5 dados generando 5 valores aleatorios uniformes en {1,...,6}.
    2. Se calcula la suma de los dados y se incrementa el contador de frecuencia correspondiente.
    3. Se incrementa el contador de frecuencia de cada cara individual.
    4. Se clasifica el resultado según las categorías de Yahtzee (tercia, póker, full house, escaleras, yahtzee).
    5. Se actualiza la media acumulada para el análisis de convergencia.
  3. Se calculan las estadísticas finales: media, desviación estándar, mínimo, máximo.
  4. Se generan las gráficas de distribución, frecuencias, probabilidades comparativas y convergencia.

3. Resultados

3.1 Funcionamiento del Juego

La aplicación implementa exitosamente el juego Yahtzee clásico con las siguientes características funcionales:

  • Lanzamiento de dados: Cada turno permite hasta 3 lanzamientos de 5 dados. Los dados generan valores con distribución uniforme en {1, 2, 3, 4, 5, 6}.
  • Retención selectiva: El jugador puede hacer clic en cualquier dado para "guardarlo" y volver a lanzar solo los dados no retenidos.
  • 13 categorías de puntuación: Sección superior (unos a seises), sección inferior (tercia, póker, full house, escaleras, yahtzee, chance). Cada categoría solo puede usarse una vez.
  • Bono de sección superior: Si la suma de la sección superior alcanza o supera 63 puntos, se otorgan 35 puntos de bonificación.
  • Alternancia de turnos: Los dos jugadores juegan alternadamente hasta completar las 13 categorías cada uno (26 turnos totales).
  • Atajos de teclado: Barra espaciadora para lanzar, teclas 1-5 para alternar la retención de cada dado.

La frecuencia individual de cada cara (1 a 6) se distribuye uniformemente alrededor del valor teórico de 16.67% (1/6). Las desviaciones observadas son estadísticamente insignificantes para N = 10,000, lo que confirma la uniformidad del generador de números aleatorios.

3.2.5 Convergencia de la Media

La gráfica de convergencia muestra cómo la media acumulada de la suma de dados se estabiliza progresivamente alrededor del valor teórico de 17.5 a medida que aumenta el número de iteraciones. Se observa que:

  • Con N = 100, la media puede desviarse significativamente (±1-2 unidades).
  • Con N = 1,000, la media se acerca a 17.5 con fluctuaciones menores.
  • Con N = 10,000, la convergencia es notable y la media se mantiene en el rango [17.3, 17.7].
  • Con N = 100,000+, la media es prácticamente indistinguible del valor teórico.

Este comportamiento es una demostración directa de la Ley de los Grandes Números, que establece que el promedio de los resultados de un gran número de ensayos aleatorios converge al valor esperado cuando el número de ensayos tiende a infinito.

4. Conclusiones

  1. Validación del método de Montecarlo: La simulación demostró que las probabilidades estimadas mediante el método de Montecarlo convergen hacia los valores teóricos exactos a medida que se incrementa el número de iteraciones. Esta convergencia es consecuencia directa de la Ley de los Grandes Números y valida la utilidad del método para estimar probabilidades de eventos complejos.

  2. Distribución uniforme verificada: Los resultados confirman que la función Math.random() de JavaScript genera valores con distribución uniforme adecuada para simulaciones estadísticas. La frecuencia de cada cara del dado se distribuyó uniformemente alrededor del 16.67% teórico.

  3. Eventos raros y tamaño de muestra: Se observó que los eventos de baja probabilidad (como el Yahtzee) requieren un número significativamente mayor de iteraciones para alcanzar estimaciones confiables. Esto subraya la importancia de seleccionar un tamaño de muestra apropiado según la precisión deseada en la estimación.

  4. Teorema del Límite Central: La distribución de la suma de cinco dados presenta una forma acampanada simétrica que se aproxima a la distribución normal, lo cual es consistente con el Teorema del Límite Central. Este resultado emerge naturalmente de la simulación sin necesidad de supuestos adicionales.

  5. Aplicabilidad práctica: El método de Montecarlo demostró ser una herramienta poderosa y flexible para analizar problemas probabilísticos. Su implementación computacional es directa y escalable, requiriendo únicamente un generador de números pseudo-aleatorios de calidad y la capacidad de ejecutar un gran número de iteraciones.

  6. Relación costo-precisión: Existe un compromiso entre el número de iteraciones (costo computacional) y la precisión de las estimaciones. La simulación permitió observar que el error relativo disminuye aproximadamente como O(1/√N), donde N es el número de iteraciones, lo cual es una propiedad fundamental del método de Montecarlo.
